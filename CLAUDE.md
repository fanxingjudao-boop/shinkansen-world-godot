# しんかんせんワールド (Godot 4 版) — Claude Code 作業指示書

このファイルは Claude Code がこのプロジェクトで作業するときに最初に読むファイルです。
作業開始前に必ず `docs/HANDOFF.md` `docs/ARCHITECTURE.md` `docs/ROADMAP.md` `docs/GODOT_NOTES.md` も読んでください。

## プロジェクトの本質

**子供にとって最高の体験を作る創作プロジェクト**です。

身内のお子さん・お孫さんが iPad で遊ぶゲームですが、「身内向け」を理由に手を抜く意味はありません。
むしろ商業制約(ライセンス・課金・利用規約)から解放されているからこそ、自由な発想で楽しさだけを追求できます。

ターゲットの子供にとって、これは「Anthropic 製の Claude が作ったゲーム」ではなく、純粋に**初めて出会う 3D ゲーム体験のひとつ**になるかもしれません。その重みを意識して作ってください。

## 技術スタック(確定事項)

- **エンジン**: Godot 4.3 以降(最新安定版を使用)
- **言語**: GDScript(C# ではない、後述)
- **レンダラー**: Compatibility(Web Export 用)
- **配信**: Web Export(PWA)を主、将来 iOS ネイティブ展開も視野
- **開発環境**: Windows 11(改善さんの環境)
- **バージョン管理**: Git + GitHub
- **ホスティング**: GitHub Pages または Vercel(改善さんが選択)

### 重要: 言語選択の経緯

当初 C# が候補に挙がったが、以下の理由で GDScript を採用しました:

1. **C# の Web Export は現状実験的**で、Godot 公式が「開発中」と明言している領域
2. C# Web Export はビルドサイズが GDScript の数倍(30〜80MB)で、iPad Safari でのロードが現実的でない
3. 改善さんの Python ベースのスキルセットと GDScript の親和性が高い
4. 将来 Unity に移植する場合でも、設計思想を共有していればコスト差は限定的

「将来 C# も使うかもしれない」考慮は、**ゲームロジックを明確にモジュール化する設計**で代替します。
言語ではなく設計の継承で対応する方針です。

### 重要: レンダラー選択の経緯

当初「ガッチリ作り込んだグラフィック(PBR、ポストエフェクト、SDFGI)」が希望でしたが、以下を考慮して中間品質に調整:

1. Godot 4 の Web Export は **Compatibility レンダラー**(WebGL2 準拠)が標準で、Forward+/Mobile の高度な機能の多くが使えない
2. SDFGI、Volumetric Fog、SSAO、Screen Space Reflection 等は Web では動かない
3. それでも以下は使える: **PBR マテリアル**、**標準シャドウ**、**Glow(部分的)**、**カスタムシェーダー**、**Particles**

つまり「**作り込み**」の方向性は、**最先端ポストエフェクト**ではなく、**手の込んだシェーダーとマテリアル設計**にシフトします。
子供向けかわいい世界では、写実性より「光って、はじけて、ふわふわする」表現の方が刺さります。これは Compatibility レンダラーでも十分実現可能です。

## 創作の方向性

### 自由な発想を歓迎します

ROADMAP に書いた以上のことをやって構いません。
むしろ、子供が「わぁっ!」と声を上げる瞬間を増やすために、自由にアイデアを出してください。

例:
- 動物たちが歌う
- 駅で買えるお団子(食べると元気度アップ)
- 電車から手を振ると沿線の人が振り返す
- 流れ星を見つけたら願い事(ランダム良いことが起きる)
- 動物の家を発見すると、その動物がそこに帰ったり出てきたり
- 季節が変わると桜が散る・雪が降る・紅葉する
- 線路の分岐で運転手になれる
- 夜にホタルが出る
- 雨上がりに虹がかかる

判断に迷うものは改善さんに確認を取ってください。

### 子供への配慮(これは厳守)

自由な発想は奨励しますが、以下の配慮は**例外なく**守ってください。

- **怖くないこと**: 暗すぎる夜、不気味な音、急に出てくる敵、ジャンプスケア、ダメージ表現、失敗・ゲームオーバー、これらすべて禁止
- **読めること**: ひらがな・カタカナのみ。漢字を出さない
- **押せること**: タッチターゲットは最小 48×48px、できれば 64×64px
- **わかること**: 何をすればよいか常に明示
- **やさしい音**: 大きすぎる音、不快な周波数、刺激的な効果音を避ける
- **時間がかかってもよい**: 制限時間禁止。失敗概念禁止
- **達成感**: できるだけたくさんの「やったね!」の瞬間を作る
- **公平性**: ランダム要素は良い結果に寄せる

これらは「うちのアルバム」で高齢者向け UI を作るときと同じ哲学です。
**ターゲットユーザーの実情に合わせた配慮は、技術的にエレガントでなくても、いつも正しい。**

### 美意識

- **色彩**: パステル基調(空色 `#7ec8f5`・草原緑 `#7ec850`・桜色 `#ff9ec0`・黄色 `#ffe066`・オレンジ `#ffa94d`)
- **タイポ**: 'Mochiy Pop One'(見出し)+ 'M PLUS Rounded 1c'(本文)
- **モーション**: バネ感のあるイージング(`ease_out_back` 等)を多用、直線的な動きは避ける
- **音楽**: 優しい正弦波・三角波中心、または穏やかな BGM 1〜2 曲

改善さんの普段の好み(ダーク・ネオン・SANKOU! cool 系)は**このプロジェクトでは適用しません**。
「子供と大人が一緒に見て、両方が良いと思う」可愛い世界を作ります。

## Godot 4 の重要な技術的注意点

詳細は `docs/GODOT_NOTES.md` を参照してください。要点のみ:

### iPad Safari 対応

- Compatibility レンダラーを使用(`project.godot` で設定)
- Web Export 時に **COEP/COOP ヘッダー**が必要(GitHub Pages では設定不可、Vercel 推奨)
- フォントは Web フォントを併用、ゲーム内 UI には Godot に埋め込み
- タッチ入力は `InputEventScreenTouch` + `InputEventScreenDrag`
- AudioContext はユーザー操作後に開始(スタートボタンで初期化)
- `100vh` 問題は HTML テンプレート側で対応(`100dvh` 使用)

### パフォーマンス

- 影は `DirectionalLight3D` の標準シャドウのみ
- SDFGI、Volumetric Fog、SSAO は使わない
- PBR マテリアルは OK だが、`metallic`、`roughness` は控えめに
- MultiMeshInstance3D で大量オブジェクト(木、花)を統合
- LOD は子供向けではあまり気にしなくて良い(モデルが単純なので)

### 過去の Three.js 版から引き継ぐ知見

- 左右の座標系: Three.js では yaw 計算でバグがあった。Godot の `Camera3D` は標準的なので問題なし
- CDN フォールバック: Godot は Web Export 時に必要なファイルを同梱するので CDN 依存はゼロ
- 単一ファイル制約: Godot ではシーン分割が前提、`docs/ARCHITECTURE.md` 参照

## セキュリティ原則(子供向け配信)

子供が遊ぶ Web 配信物として、**プライバシーと安全を最優先**します。以下は子供への配慮(怖くない等)と同格の「例外なく厳守」事項です。

### 厳守事項

- **個人情報を一切収集・保存しない**。セーブは進捗の識別子(slug:`hayabusa` 等)と数値のみ(`scripts/world/save_system.gd`)。氏名・年齢・位置・連絡先・端末情報などは扱わない。
- **外部広告・アナリティクス・トラッキングを導入しない**(Google Analytics・Vercel Analytics・各種 SDK 等)。コードにも HTML テンプレートにも入れない。
- **外部リンク・SNS 連携・テキスト入力欄を作らない**(`LineEdit`/`TextEdit`/`OS.shell_open`/`HTTPRequest` を使わない)。子供が意図せず外部へ出る経路を作らない。
- **外部リソースへの通信を発生させない**。フォント等は CDN ではなく**プロジェクトに同梱**(`assets/fonts/`)。`self`(同一オリジン)以外への fetch/接続を増やさない。
- **セキュリティヘッダーを維持する**(`vercel.json`):CSP・Permissions-Policy・X-Frame-Options(DENY)・Referrer-Policy(no-referrer)・X-Content-Type-Options(nosniff)・COEP/COOP。
  - CSP は Godot Web が動く範囲(`script-src` に `'wasm-unsafe-eval' 'unsafe-inline'`、`worker-src blob:` 等)で維持。**ヘッダー変更時はデプロイ後にゲームが起動するか必ず確認**(CSP は起動を壊しうる唯一のヘッダー)。
- **検索エンジンに載せない**:`export/web/robots.txt`(Disallow: /)+ `noindex`(`web/template.html` の meta と `vercel.json` の X-Robots-Tag)を維持。
- **新機能追加時、それが子供の安全・プライバシーに影響しないか必ず確認する**。迷ったら改善さんに確認。

### 判断の早見

新機能で「入力欄/外部通信/外部リンク/個人データ/トラッキング」のいずれかが必要になったら、**実装前に改善さんへ確認**(原則は不要なはず)。`vercel.json`・`robots.txt`・`web/template.html` を触ったら、デプロイ後に①ゲームが起動する②文字が出る③外部通信が増えていない(開発者ツールのネットワークタブ)を確認する。

## 創作判断のヒント

子供向け UX で迷ったときの判断軸:

1. **怖くないか?** — 怖いなら却下
2. **難しすぎないか?** — 3 歳児が理解できないなら却下
3. **嬉しいか?** — 達成感・発見感がないなら微妙
4. **公平か?** — 子供が悲しむ可能性があるなら却下

迷ったら、改善さんに確認を取ってください。
特に「子供にとって楽しいか」「怖くないか」「分かるか」は、現場感覚を持つ改善さんに委ねるのが最善です。
